Frequently Asked Questions

1) What if no valid base can be formed?

If none of the 3-card base combinations sum to a multiple of 10, that hand has no valid arrangement under this ruleset. 計算器會直接顯示「找不到合法排法」。

2) Is A always AS in input?

No. You can input plain A without suit. For direct 牛冬菇 recognition, the point card must be explicit AS with J/Q/K. If suit is unknown, the calculator can show an assumed Ace outcome separately.

3) Which is stronger: 孖寶 or 10點?

In this implementation, multiplier is prioritized. 孖寶 has multiplier 3 and 10點 has multiplier 2, so pair often ranks above 10-point outcomes.

4) Why does the calculator sometimes choose a split with lower-looking points?

Because the ranking is not based on points alone. It compares multiplier first, then points, then kicker. A lower visible number can still be the stronger full outcome.

5) Should I always manually test all 3/6 swaps?

You can, but it is error-prone in fast play. This calculator already checks every swap variant and returns the best legal result, which is faster and safer.
